# 需求分析器 (requirement_analyzer)

## 功能概述

作为资深系统架构师，智能分析用户需求并设计完整模块架构。

## 核心能力

1. **需求解构**：识别核心业务实体、业务流程、数据关系
2. **模块设计**：智能确定模块数量和功能划分
3. **字段推导**：为每个模块设计详细字段（类型、关联、字典）
4. **架构建议**：模块拆分、关联设计、扩展性优化

## 输出内容

- 模块数量和架构设计
- 每个模块的详细字段清单
- 数据类型和关联关系
- 字典需求和类型定义
- 模块间关系图

## 适用场景

- 需求描述不完整，需要智能补全
- 复杂业务系统的架构设计
- 需要专业的数据库设计建议
- 快速搭建生产级业务系统

## 调用示例

```
需求: "我要做一个猫舍管理系统，录入猫的信息，记录每只猫每天的活动"
```

输出：

- 模块1: 猫咪档案（基本信息、健康状态、入舍时间）
- 模块2: 活动记录（日期、活动类型、描述、关联猫咪）
- 字典: 猫品种、健康状态、活动类型
